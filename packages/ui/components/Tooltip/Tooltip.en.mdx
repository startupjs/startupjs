## Usage

- content: hidden part, can be string or jsx
- children: visible part

## Tooltip with pressable elements

If you need to wrap a pressable element in a tooltip in a mobile app, the `onPress` property must be in the top component of children. Note that the tooltip will overwrite the `onLongPress` and `onPressOut` properties.

- wrong

```jsx
<Tooltip
  content='Some content'
>
  <View>
    <Button
      onPress={onPressHandler}
    >
      Button...
    </Button>
  </View>
</Tooltip>
```

- right

```jsx
<Tooltip
  content='Some content'
>
  <Button
    onPress={onPressHandler}
  >
    Button...
  </Button>
</Tooltip>
```

## Example with Button

```jsx example
const [pressed, setPressed] = useState('')

return (
  <View>
    <View 
      style={{
        width: 100,
        marginBottom: 16
      }}
    >
      <Tooltip
        content='Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.'
      >
        <Button
          onPress={() => setPressed('onPress')}
          onLongPress={() => setPressed('onLongPress')}
        >
          Button...
        </Button>
      </Tooltip>
    </View>
    {!!pressed && (
      <Alert
        variant='success'
        label={`${pressed} event`}
        onClose={() => setPressed('')}
      />
    )}
  </View>
)
```
