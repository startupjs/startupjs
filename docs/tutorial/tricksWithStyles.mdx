# Tricks with styles

StartupJS styling is powered by CSSX. Write local component styles in `css`,
`styl`, or Pug `style` blocks, and use `StartupjsProvider style` for global
theme tokens and component defaults.

## Prefer CSS variables for reusable values

Put app-wide tokens in provider CSS:

```jsx
import { css, StartupjsProvider } from 'startupjs'

const appStyle = css`
  :root {
    --primary: oklch(0.58 0.22 260);
    --color-primary: var(--primary);
    --Button-radius: 9999px;
  }
`

<StartupjsProvider style={appStyle}>
  <App />
</StartupjsProvider>
```

Use those values from normal component CSS:

```jsx
css`
  .button {
    background-color: var(--color-primary);
    border-radius: var(--Button-radius);
  }
`
```

## Spacing and units

New code should prefer `rem`, `calc()`, and theme variables:

```css
.card {
  padding: 1rem;
  gap: calc(var(--spacing) * 2);
}
```

The old `u` unit still compiles for migration (`1u = 8px`), but new styles
should use standard CSS units.

## Responsive styles

Use CSSX custom media aliases:

```css
@media (--breakpoint-tablet) {
  .card {
    padding: 1.5rem;
  }
}

@media (--breakpoint-desktop) {
  .card {
    padding: 2rem;
  }
}
```

StartupJS UI provides default `mobile`, `tablet`, `desktop`, and `wide`
breakpoint aliases through CSSX.

## Theme-specific styles

Use theme media aliases for normal styles that should change in dark mode:

```css
.card {
  box-shadow: var(--shadow-sm);
}

@media (--theme-dark) {
  .card {
    box-shadow: none;
    border-color: var(--color-border);
  }
}
```

Define theme variables in `:root` and `:root.dark`:

```css
:root {
  --background: oklch(1 0 0);
  --color-background: var(--background);
}

:root.dark {
  --background: oklch(0.145 0 0);
}
```

## Component defaults and parts

Provider CSS can customize all instances of a themed component:

```css
Button {
  min-width: 10rem;
}

Button:part(text) {
  font-weight: var(--font-weight-semibold);
}
```

Use component variables first, then tag selectors and `:part()` selectors. Use
per-instance `style` and `*Style` props only for one-off exceptions.

## Values in JavaScript

Keep visual values in CSS when possible. When JavaScript needs a resolved token,
use CSSX hooks re-exported by StartupJS:

```jsx
import { useCssColor, useCssVariable } from 'startupjs'

function StatusDot () {
  const color = useCssColor('success')
  const size = useCssVariable('--StatusDot-size', '0.5rem')

  return <View style={{ width: size, height: size, backgroundColor: color }} />
}
```
