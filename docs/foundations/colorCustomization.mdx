# Color customization

StartupJS uses CSSX provider styles for app-wide color customization. In a
StartupJS app, pass a CSSX sheet to `StartupjsProvider style`. If
`startupjs-ui` is installed, its default Tailwind tokens, shadcn semantic
tokens, and component tokens are layered automatically, and your provider style
overrides them.

```jsx
import { css, StartupjsProvider } from 'startupjs'

const appStyle = css`
  :root {
    --primary: oklch(0.58 0.22 260);
    --primary-foreground: oklch(0.98 0.02 260);
    --color-primary: var(--primary);
    --color-primary-foreground: var(--primary-foreground);
  }
`

export default function RootLayout ({ children }) {
  return (
    <StartupjsProvider style={appStyle}>
      {children}
    </StartupjsProvider>
  )
}
```

## Recommended override order

1. Override semantic variables in `:root`.
2. Add dark or named-theme overrides in `:root.dark` or `:root.<theme>`.
3. Override component variables in `:root`.
4. Use component tag selectors and `:part()` for global component defaults.

Define CSS variables only in `:root` or `:root.<theme>` provider blocks.
Component selectors are for actual styles, not for theme variable declarations.

## Semantic colors

StartupJS UI follows the shadcn-style token pattern. App themes usually override
semantic variables such as `--primary`, `--background`, and `--border`, then map
them to Tailwind-compatible `--color-*` variables.

```jsx
const appStyle = css`
  :root {
    --background: oklch(1 0 0);
    --foreground: oklch(0.145 0 0);
    --primary: oklch(0.55 0.2 250);
    --primary-foreground: oklch(0.98 0.02 250);
    --border: oklch(0.9 0 0);

    --color-background: var(--background);
    --color-foreground: var(--foreground);
    --color-primary: var(--primary);
    --color-primary-foreground: var(--primary-foreground);
    --color-border: var(--border);
  }
`
```

Prefer semantic tokens over raw palette variables in app and component styles.
That keeps brand changes localized to the theme layer.

## Light and dark themes

The default provider theme is `auto`, so CSSX uses the device color scheme when
a dark theme exists. Define light/default values in `:root` and dark overrides
in `:root.dark`:

```jsx
const appStyle = css`
  :root {
    --background: oklch(1 0 0);
    --foreground: oklch(0.145 0 0);
    --color-background: var(--background);
    --color-foreground: var(--foreground);
  }

  :root.dark {
    --background: oklch(0.145 0 0);
    --foreground: oklch(0.985 0 0);
  }
`
```

Force a theme with `StartupjsProvider theme`:

```jsx
<StartupjsProvider theme='dark' style={appStyle}>
  <App />
</StartupjsProvider>
```

Use `theme='default'` or `theme='light'` to disable automatic dark selection.

## Component variables and parts

StartupJS UI components expose component-specific variables, component tag
selectors, and semantic parts.

```css
:root {
  --Button-radius: 9999px;
  --TextInput-border-color-focused: var(--color-primary);
}

Button {
  min-width: 10rem;
}

Button:part(text) {
  font-weight: var(--font-weight-semibold);
}
```

Each component docs page lists its available parts and component variables.

## Colors in JavaScript

Prefer CSS for styling. When JavaScript needs a resolved color token, use the
CSSX bridge re-exported by StartupJS:

```jsx
import { useCssColor } from 'startupjs'

function StatusDot () {
  const color = useCssColor('success')
  return <View style={{ backgroundColor: color }} />
}
```

`useCssColor('primary')` resolves `var(--color-primary)`.
`useCssColor('primary-foreground')` resolves
`var(--color-primary-foreground)`. Pass an exact `var(...)` expression when you
need a custom variable.
