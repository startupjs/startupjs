# CSS values in JavaScript

Most visual values should stay in CSS. Use CSS variables in `css` or `styl`
templates, and read them in JavaScript only when a non-CSSX API needs the final
value.

## Read theme variables

Use `useCssVariable()` for provider-aware CSS variable reads:

```jsx
import { useCssVariable } from 'startupjs'

function Chart () {
  const gap = useCssVariable('--spacing', '0.25rem')
  return <ChartCanvas gridGap={gap} />
}
```

The hook resolves nested `var()` references and returns React Native-friendly
values. For example, `16px` becomes `16`, while percentages remain strings.

## Read color tokens

Use `useCssColor()` for colors:

```jsx
import { useCssColor } from 'startupjs'

function StatusDot () {
  const color = useCssColor('success')
  return <View style={{ backgroundColor: color }} />
}
```

`useCssColor('primary')` resolves `var(--color-primary)`. Pass a full
`var(...)` expression for custom variables:

```jsx
const color = useCssColor('var(--User-color-online)')
```

## Static exports

CSSX still supports static `:export` values for build-time constants, but do
not use it as a theme system. Runtime themes should use provider variables and
CSSX hooks.

```jsx
css`
  :export {
    iconSize: 24;
  }
`
```

Use static exports only for values that do not change with theme, provider, or
screen size.
