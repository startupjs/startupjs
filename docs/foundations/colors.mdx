# Colors

StartupJS color usage is CSSX-first. Use semantic CSS variables in styles and
CSSX color hooks only when JavaScript needs a resolved value.

For theme overrides, see [Color customization](./colorCustomization).

## Token layers

When `startupjs-ui` is installed, its plugin layers:

1. Tailwind-compatible raw tokens, such as spacing, radius, typography, and
   palette variables.
2. shadcn-style semantic tokens, such as `--primary`, `--background`,
   `--foreground`, `--border`, and `--muted`.
3. StartupJS UI component tokens, such as `--Button-radius`.
4. Your `StartupjsProvider style` overrides.

Bare StartupJS does not install these tokens automatically. They are added when
the UI package is used, or when you explicitly import CSSX theme assets.

## Semantic color variables

Use semantic `--color-*` variables in component and app CSS:

```jsx
import { css } from 'startupjs'

css`
  .card {
    background-color: var(--color-card);
    border-color: var(--color-border);
  }

  .title {
    color: var(--color-foreground);
  }

  .muted {
    color: var(--color-muted-foreground);
  }
`
```

Common semantic colors include:

| Token | Typical use |
| --- | --- |
| `--color-background` | app/page background |
| `--color-foreground` | default text |
| `--color-card` | card and panel background |
| `--color-card-foreground` | text on cards |
| `--color-primary` | primary actions and emphasis |
| `--color-primary-foreground` | text/icons on primary surfaces |
| `--color-secondary` | secondary surfaces |
| `--color-muted` | muted surfaces |
| `--color-muted-foreground` | secondary text |
| `--color-border` | borders and dividers |
| `--color-error` | destructive/error states |
| `--color-success` | success states |
| `--color-warning` | warning states |
| `--color-info` | informational states |

Prefer these semantic tokens over direct palette values. Override the semantic
source variables, such as `--primary` or `--background`, to change a theme.

## Color props

StartupJS UI color props accept token names, raw CSS colors, or exact `var(...)`
expressions:

```jsx
<Button color='primary'>Save</Button>
<Tag color='warning'>Warning</Tag>
<Loader color='var(--color-muted-foreground)' />
```

Token names resolve through `--color-*`. For example, `primary` resolves through
`--color-primary`, and `primary-foreground` resolves through
`--color-primary-foreground`.

## Using colors in JavaScript

Use `useCssColor()` when JS needs the final React Native-friendly color value:

```jsx
import { useCssColor } from 'startupjs'

function StatusDot () {
  const color = useCssColor('success')
  return <View style={{ backgroundColor: color }} />
}
```

For visual styling, prefer CSS variables directly in CSS. Use the JS bridge for
non-CSSX props, canvas/chart integrations, or logic that genuinely needs the
computed color.
